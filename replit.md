# CannonBall

## Overview
CannonBall is a full-stack web application for comprehensive automation pipeline management. It guides users from idea capture to deployment of AI-generated automation packages, aiming to streamline development and deployment. Key features include an AI-first approach, a role-based shell, a Kanban board, a three-panel workspace with live AI chat, and a visual process map engine. The platform focuses on automating interactions and generating deployable UiPath automation solutions.

## User Preferences
The agent should prioritize an AI-first interaction style, proactively engaging users and providing guidance throughout the automation pipeline. It should facilitate an iterative development process, allowing for dynamic adjustments such as moving ideas to earlier stages if requirements change. The agent should ask for confirmation or approval before making significant changes, especially regarding document generation, process map approvals, and deployment actions. Communication should be clear and concise, with real-time feedback on progress, such as stage transitions and deployment statuses. The agent must adhere to structured outputs for documents and deployment artifacts. **Reduce redundant code**: Always consolidate duplicated logic into single canonical functions. When implementing shared behavior, create one source of truth and have all callers use it. Update all references across the app when consolidating — do not leave orphaned implementations that could cause discrepancies.

## System Architecture
The application uses a modern web stack for scalability and an intuitive user experience.

-   **Frontend**: React with Vite, Tailwind CSS, and shadcn/ui. It uses React Flow for interactive process mapping and wouter for client-side routing, featuring a responsive UI, dark mode, and a distinct color palette.
-   **Backend**: Express.js server handling API requests and session-based authentication.
-   **Database**: PostgreSQL with Drizzle ORM.
-   **AI Integration**: Anthropic Claude via Replit AI Integrations for streaming SSE chat responses.
-   **Authentication**: Session-based authentication supporting demo users and role-switching.

**Key Features and Design Choices:**

-   **UI/UX**: Responsive sidebar, top navigation, and a resizable three-panel workspace displaying stage progress, a live React Flow process map, and the AI chat.
-   **UiPath Agent Awareness**: Full pipeline support for automation type evaluation (RPA vs Agent vs Hybrid). During feasibility, the AI evaluates processes against a decision framework (structured vs unstructured, rule-based vs judgment-based, volume, systems, error patterns, cost) and outputs an `[AUTOMATION_TYPE: rpa|agent|hybrid | rationale]` tag. The chosen type flows through the entire downstream pipeline: process map (agent-task/agent-decision nodes with purple styling), PDD (automation approach section), SDD (agent architecture + agent artifacts), XAML generation (InvokeAgent activities), and deployment (agent config assets, knowledge bases, prompt templates provisioned to Orchestrator). DB fields: `ideas.automationType`, `ideas.automationTypeRationale`, `ideas.agentConfig`.
-   **Process Map Engine**: Supports custom node types (Start/End, Task, Decision, Agent-Task, Agent-Decision) with DAG-based layout, confidence scoring, inline editing, context menus, and an approval workflow. Agent nodes render with distinct purple/AI styling. Dynamic updates from AI's `[STEP: 1.0 Name ...]` tags ensure deterministic step-number-based edge resolution.
-   **Automated Stage Transitions**: An engine automatically transitions ideas across 10 pipeline stages based on predefined criteria, with comprehensive audit logging.
-   **Document Generation**: Automates generation of version-controlled Process Design Documents (PDD) and Solution Design Documents (SDD) post-approvals. SDD generation is conversational and auto-triggered on PDD approval. Exported documents include visual process map images (PNG) rendered server-side.
-   **AI-Enriched UiPath Package Generation & Deployment**: Generates near-production-ready UiPath packages with AI-powered enrichment, including multi-activity sequences, system-specific selectors, context-aware variable names, and real property values from SDD artifacts. It adheres to UiPath Package Explorer compliance, includes traceability comments, supports REFramework, and enhances `Config.xlsx` and `InitAllSettings.xaml`. All 14 artifact types are generated at production quality (queues, assets, machines, triggers, storage buckets, environments, robot accounts, Action Center, Document Understanding, test cases, test sets, requirements, test data queues). XAML activities include standard properties, production-grade selectors, and `sap2010:Annotation.AnnotationText` on every activity with business context, step traceability, and error strategy. Conversational deployment to UiPath Orchestrator with live status streaming is supported.
-   **Workflow Analyzer Compliance Engine** (`server/workflow-analyzer.ts`): Deterministic static analysis checking 11 Workflow Analyzer rules (ST-NMG naming, ST-DBP best practices, ST-USG usage, ST-SEC security) with auto-correction for variable/argument renaming, empty catch blocks, missing start/end logs. Exports `analyzeXaml()`, `analyzeAndFix()`, `formatAnalysisReportMarkdown()`, `generateAnnotationText()`, `generateArgumentValidationXaml()`. Integrated via `compliancePass()` in `uipath-integration.ts`.
-   **UiPath Naming Convention Enforcement**: `enforceVariableName(name, type)` → `{str|int|bool|dt|dbl|dec|obj|ts|drow|qi|qid}_{PascalCase}`; `enforceArgumentName(name, direction)` → `{in|out|io}_{PascalCase}`. Applied throughout the XAML generation pipeline.
-   **Argument Validation at Entry Points**: Every invokable workflow gets a "Validate Arguments" Sequence as first child, checking required InArgument/InOutArgument values with BusinessRuleException throws, Trace-level logging (with credential masking), and annotation.
-   **Three-Tier Developer Handoff Guide (DHG)**: Restructured with AI-first philosophy — Tier 1 (AI Completed, no human action), Tier 2 (Smart Defaults, review recommended), Tier 3 (Human Required, minimal). Includes readiness score, Workflow Analyzer compliance report, embedded code review rubric, and sign-off template. Agent Tier 1 includes provisioned agent artifacts; Tier 2 includes guardrail/escalation review; Tier 3 includes knowledge base content population, prompt tuning, agent testing.
-   **UiPath Integration Layer**: Manages UiPath's multi-resource token architecture (5 independent cached tokens for Orchestrator, Test Manager, Document Understanding, Platform Management, Data Service) with proactive refresh and 401 auto-retry. Includes a typed Orchestrator API client with exponential backoff. Supports full Test Manager lifecycle, idempotent create-or-update for artifacts, and robust Action Center catalog creation attempts. Chat route ensures service probes run regardless of pipeline stage and strips superseded SDD/PDD messages from chat history.
-   **Unified Probe Architecture**: `probeAllServices()` is the single source of truth for platform service availability (60s TTL cache). `probeServiceAvailability()` and `getPlatformCapabilities()` are thin wrappers. `verifyUiPathScopes()` consumes cached probe for service flags. `preflightInfraProbe()` in deploy accepts `ServiceAvailabilityMap` for AC/TM flags, only fetches folder-specific data (machines, robots, sessions). `detectAvailableRuntimeType()` reuses pre-fetched session data from infra probe. `clearProbeCache()` called on all config/connection changes.
-   **Consolidated Streaming Progress Indicator**: A single `StreamingProgressIndicator` component (`workspace.tsx`) handles all three wait-state contexts: `"thinking"` (three pulsing dots for normal chat), `"doc"` (CannonballSpinner with real-time section names from server `docProgress` SSE events during PDD/SDD generation), and `"deploy"` (CannonballSpinner with live deployment step text from `deployStatus` SSE events). Replaces the former separate `ThinkingIndicator`, `DocProgressIndicator`, and deploy-status fake-message rendering. Server emits `docProgress.started` and `docProgress.section` events as the AI generates document sections, so the client shows "Now writing: Executive Summary" etc. in real-time. Falls back to timer-based step names (`DOC_PROGRESS_STEPS`) if no server events arrive.
-   **LLM-Based Intent Classification**: Before the main AI streaming call, a fast lightweight classification call (`max_tokens: 20`) determines user intent: `PDD`, `SDD`, `PDD_SDD`, `DEPLOY`, or `CHAT`. This replaces brittle regex-based detection and correctly handles conversational affirmatives ("yes go ahead"), explicit requests ("regenerated the SDD please"), and combined doc requests. The server is the single source of truth for indicator activation — the client no longer does regex-based doc detection. For `PDD_SDD` flows, the indicator automatically switches from PDD to SDD when the in-stream `[DOC:SDD:` tag is detected.
-   **Artifact Upsert on Re-deployment**: Deployment uses create-or-update (upsert) for all artifact types. Queues update schema/retries/description via PUT. Assets update values (except Credentials for security). Triggers update cron/runtime/timezone via PUT. Machines update slots/description. Storage buckets update description. Status `"updated"` shown in UI with cyan indicator distinct from green "Created" and blue "Exists". **PUT-404 Fallback**: When a PUT update returns 404 (cross-folder OData filter mismatch — artifact found in a different folder than the configured one), the system falls back to POST (create) in the current folder instead of reporting "failed". Safe because UiPath enforces unique names per folder, so POST returns 409 if it already exists. Affects all provisioning functions: queues, assets, machines, storage buckets, triggers.
-   **Integration Status Bar**: Displays live UiPath connection status, robot count, pending Action Center tasks, latency, and last provisioning decisions.
-   **Enhanced Diagnostics & Live Operations UI**: Settings tab with diagnostics checklist and a live operations panel.
-   **File Upload Content Extraction**: Server-side extraction from DOCX, PDF, XLSX, TXT, and CSV for AI chat context.
-   **Admin & Review Panels**: For CoE review and Admin tasks (user management, audit logs, system configuration).
-   **Role-Based Access**: Authorization enforced on process map and document routes based on user ownership and roles.
-   **Multi-Orchestrator Connection Management**: Supports storing and managing multiple UiPath Orchestrator connection profiles, with auto-migration of existing credentials.
-   **JSON Sanitization**: AI-generated JSON is sanitized and errors are recovered to ensure robust parsing.
