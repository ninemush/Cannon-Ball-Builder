SHARED UTILITY — sendEvent helper
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

If a sendEvent helper does not already exist 
in chat/routes.ts, create one that handles 
serialisation, error handling, and flushing:

  function sendEvent(
    res: Response, 
    data: Record<string, unknown>
  ): void {
    try {
      if (res.writableEnded) return
      res.write(
        `data: ${JSON.stringify(data)}\n\n`
      )
      if (typeof (res as any).flush === 
          'function') {
        (res as any).flush()
      }
    } catch (e) {
      // client disconnected — ignore
    }
  }

Replace all res.write('data: ...') calls in 
chat/routes.ts with sendEvent(res, ...) calls. 
This eliminates the buffering issue and adds 
resilience against client disconnects throughout.
