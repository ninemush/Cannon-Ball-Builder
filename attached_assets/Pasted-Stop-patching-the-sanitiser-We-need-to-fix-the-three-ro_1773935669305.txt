UPSTREAM FIX 3 — Catalog validation 
before rendering any activity tag

In the template builder or the activity 
rendering function, before emitting any 
XML element with a ui: prefix, validate 
that the activity name exists in the 
catalog. If it does not exist:
  - Emit a ui:Comment instead
  - Log a warning with the unknown 
    activity name
  - Never emit a tag with a made-up 
    or TODO_ name

This is the real fix for the TODO_ tag 
issue. The sanitiser is treating the 
symptom. The cause is the generator 
emitting tags that were never validated 
against the catalog.

PROMPT FIX — Reinforce bracketing and

"ABSOLUTE RULES for expressions:

3. Never use TODO_, PLACEHOLDER_, or 
   any invented name as an activity

   must exactly match a name from the 
   ACTIVITY TEMPLATES section. If you
