Quote-Safe XML Attribute Emission and Repair-Before-Stub

What & Why
Recent forensic analysis shows that many business workflows are being destroyed not because the business intent is absent, but because emitted XAML contains mechanically repairable XML well-formedness defects, especially raw quote characters inside expression-bearing attributes such as LogMessage.Message. The current pipeline often responds by stubbing or blocking the workflow instead of repairing the attribute value and continuing.

This task makes quote-safe emission a first-class requirement and changes the handling policy from "stub on repairable malformed-quote cases" to "repair deterministically before stubbing."

Goals
1. Eliminate raw unescaped quote corruption in XML/XAML attribute values.
2. Repair mechanically fixable malformed-quote issues before any workflow is stubbed.
3. Preserve business workflows that are otherwise structurally sound.
4. Record clear before/after repair diagnostics for trust and traceability.

Done looks like
- No emitted XAML attribute contains raw quote characters that make the XML not well-formed.
- Expression-bearing attributes such as LogMessage.Message are serialized with correct XML escaping.
- If malformed-quote issues are detected post-emission and can be repaired deterministically, they are repaired in-place before any stub/block decision.
- Workflows are stubbed only when quote repair cannot restore a well-formed and semantically valid attribute.
- A structured diagnostics artifact records all quote repairs and all quote-repair failures.
- Existing package-mode completeness policy still applies. Repairable quote issues no longer destroy otherwise valid workflows.

Out of scope
- Broad mail-family/source-of-truth redesign
- Catalog/filter alignment
- General parity/freeze changes
- Deep business-logic invention
- Reworking the entire decomposition strategy

Tasks

1. Audit quote-sensitive emission paths
Audit all expression-bearing attribute emission paths, especially:
- LogMessage.Message
- Assign.Value when emitted as an attribute
- any literal/vb_expression attribute serializers
- any tree assembler or AI-enricher path that may pass through LLM-generated content without XML escaping
Do not assume existing escape logic is sufficient just because some paths already call escapeXml().

2. Harden quote-safe attribute serialization
Ensure all attribute emission paths use the correct XML escaping for embedded quote characters while preserving intended VB.NET expression semantics.
Pay special attention to paths where:
- LLM-generated strings arrive pre-quoted
- attributes are reconstructed after normalization
- content passes through tree assembly or enrichment before final XAML emission

3. Add repair-before-stub for malformed-quote defects
Where malformed-quote or XML-wellformedness failures are caused by quote corruption:
- attempt deterministic in-place quote repair first
- re-run well-formedness validation after repair
- continue the workflow if repair succeeds
Only stub/block if repair fails or if safe semantics cannot be preserved

4. Intercept all relevant stub/block decision points
Do not treat stubbing as a single decision point.
Add the repair-before-stub behavior at all relevant locations where malformed-quote defects can currently lead to stub/block outcomes, including:
- gap-analyzer / malformed-quote detection
- compliance/normalization decision paths
- final validation/package assembly decision paths
- archive gate / post-assembly integrity decision paths
If a centralized interception point is possible, use it. Otherwise wire the behavior consistently at each stage.

5. Restrict repair scope to deterministic cases
Repair only when the intended attribute value can be preserved exactly by XML escaping or equivalent deterministic transformation.
Do not guess business logic.
If multiple possible repairs exist, reject rather than guessing.

6. Emit diagnostics
Add:
quoteRepairDiagnostics
with per repair attempt:
- file
- workflow
- attributePath
- originalValue
- repairedValue
- repairApplied: true/false
- repairReason
- repairFailedReason
- savedFromStub: true/false
- packageFatal: true/false

Also include summary counts:
- totalMalformedQuoteFindings
- totalQuoteRepairsApplied
- totalQuoteRepairsFailed
- totalWorkflowsSavedFromStub
- totalWorkflowsStillStubbedAfterRepairAttempt

7. Add active-path proof
Emit a lightweight trace proving that repaired content, when repair succeeds, is the exact content used by the downstream stage that would otherwise have stubbed the workflow.
For each repaired workflow include:
- stageWhereDetected
- stageWhereApplied
- preRepairHash
- postRepairHash
- downstreamConsumedRepairedVersion: true/false

In debug/test mode, fail if a repair is recorded but a later stage still consumes the unrepaired version.

8. Add focused regression tests
Cover:
- LogMessage.Message with embedded quotes is emitted as well-formed XML
- deterministic post-emission repair rescues a workflow that would previously be stubbed
- workflows are not stubbed for repairable quote-only defects
- unrecoverable malformed attributes still fail honestly
- repair diagnostics capture before/after values
- debug/test mode fails if repaired content is not what downstream consumed
