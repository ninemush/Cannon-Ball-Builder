Add a real-time streaming log to the package generation pipeline so users see the AI's decisions as they happen, rather than waiting on a static spinner.

BACKGROUND
The package generation pipeline already has clear stage boundaries: SDD approval → LLM spec generation → AI XAML enrichment → XAML generation + compliance → quality gate → NuGet packaging → DHG generation. Each stage is a natural story beat. This task wires a live narrative stream to those stages so the frontend can show a continuously updating log — similar to how Replit's agent or UiPath Autopilot streams its reasoning.

SOLUTION — BACKEND

1. Add a PipelineEventEmitter to uipath-pipeline.ts. It should emit typed events:
   type PipelineEvent = {
     stage: string;           // e.g. 'xaml_enrichment'
     status: 'started' | 'completed' | 'warning' | 'failed';
     summary: string;         // one human-readable line, e.g. "Detected REFramework pattern — using full transactional scaffold with retry logic"
     durationMs?: number;     // populated on completed/failed
     detail?: string;         // optional extra info for warnings/failures
     timestamp: number;
   }

2. Emit a started event at the beginning of each stage and a completed (or warning/failed) event at the end. The summary on the completed event should reflect the actual decision made — not a generic "done" message. Examples:
   - "Resolved 4 activity dependencies — all canonical names confirmed"
   - "Quality gate passed — 0 errors, 2 warnings (see package details)"
   - "Generated 847-line REFramework XAML in 3.2s"

3. Add a new SSE endpoint: GET /api/ideas/:ideaId/package/stream
   - Accepts an ideaId
   - Returns text/event-stream
   - Streams PipelineEvents as they are emitted during generation
   - Closes the stream when the pipeline emits a terminal event (completed, failed)
   - If the package is already built and cached, emit a single synthetic event summarising the cached result and close immediately

4. If SSE is already in use elsewhere in the codebase, follow the existing pattern. If not, use the standard Node.js res.write('data: ...\n\n') approach with res.setHeader('Content-Type', 'text/event-stream').

SOLUTION — FRONTEND

5. In the package generation view, replace the existing spinner/progress bar with a streaming log panel. The panel should:
   - Show a pulsing dot next to the currently active stage name
   - Stream each summary line in as it arrives, with a subtle fade-in (no typewriter character-by-character effect — line-by-line is cleaner)
   - Show elapsed time next to each completed stage
   - Highlight warnings in amber, failures in red
   - Auto-scroll to the latest event
   - On completion, transition smoothly to the package ready state without a page reload

6. The log panel should have a fixed max-height with internal scroll so it does not push page content around as it grows.

7. If the EventSource connection drops and the build is still in progress, show a "Reconnecting..." state and attempt to reconnect up to 3 times before showing an error.

8. All existing tests must still pass. Add a test that verifies a PipelineEvent is emitted for each stage.
