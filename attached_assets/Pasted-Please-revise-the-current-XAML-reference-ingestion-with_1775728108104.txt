Approval rubric
An activity should be marked emissionApproved: true only when all of these are true:
1. Schema truth is trusted
   - source is trusted, ideally "dll-extract"
   - propertiesComplete = true
   - property schema is complete enough for strict validation
   - the activity is not from a known failed/missing DLL package unless covered by another trusted source
2. Namespace/package mapping exists
   - prefix, xmlns, clrNamespace, and assembly are known
   - runtime namespace maps, prefix-to-package maps, and quality-gate prefix allowlists are aligned
   - if the package uses sub-namespaces, those are modeled too
3. The activity is not policy-excluded
   - not deprecated with a preferred replacement
   - not internal framework plumbing
   - not an unsupported generic backtick class
   - not a special-runtime-context activity we are not ready to broadly support
4. At least one minimal generation path exists
   - deterministic generator exists and is aligned, or
   - dynamic template path works, or
   - another validated emission path exists
5. A smoke validation passes
   - namespace resolves
   - dependency analyzer finds the package
   - emission gate passes
   - strict property validation does not falsely strip valid properties
   - no obvious structured-object leakage or namespace/import failure occurs
