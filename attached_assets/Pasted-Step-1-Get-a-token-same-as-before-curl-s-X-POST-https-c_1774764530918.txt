Step 1 — Get a token (same as before):

curl -s -X POST "https://cloud.uipath.com/identity_/connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET&scope=OR.Tasks OR.Tasks.Read OR.Tasks.Write OR.Default" \
  | python3 -m json.tool
Copy the access_token.

Step 2 — Create a task catalog using the correct OData action endpoint:

curl -s -w "\nHTTP_STATUS: %{http_code}\n" -X POST \
  "https://cloud.uipath.com/uipatezkzunj/DefaultTenant/orchestrator_/odata/TaskCatalogs/UiPath.Server.Configuration.OData.CreateTaskCatalog" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -H "X-UIPATH-OrganizationUnitId: 2122822" \
  -d '{"Name": "CannonBall_AC_Test_ODataAction", "Description": "Testing correct OData action endpoint"}'
Step 3 — Verify it exists:

curl -s -w "\nHTTP_STATUS: %{http_code}\n" \
  "https://cloud.uipath.com/uipatezkzunj/DefaultTenant/orchestrator_/odata/TaskCatalogs?\$filter=Name%20eq%20'CannonBall_AC_Test_ODataAction'&\$top=1" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-UIPATH-OrganizationUnitId: 2122822"
Step 4 (optional) — Clean up by deleting it:

curl -s -w "\nHTTP_STATUS: %{http_code}\n" -X DELETE \
  "https://cloud.uipath.com/uipatezkzunj/DefaultTenant/orchestrator_/odata/TaskCatalogs(CATALOG_ID_FROM_STEP_2)" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-UIPATH-OrganizationUnitId: 2122822"
(Replace CATALOG_ID_FROM_STEP_2 with the Id value from the creation response. Note: the docs say catalogs can't be deleted, but Swagger shows a DELETE endpoint — worth trying.)
