#62 - Fix XAML XML validation require error blocking package downloads

What & Why
The validateXamlContent function in xaml-generator.ts uses require("@xmldom/xmldom") inline. Since the project runs as ESM, require() is not available, causing xml-wellformedness validation to fail with "require is not defined" for every XAML file. This incorrectly marks valid files as failed and blocks UiPath package download.

This is a regression and should be fixed as a production bug, not treated as a one-off local workaround.

Done looks like
- UiPath package downloads complete successfully from the right-side panel without "require is not defined"
- XML well-formedness validation still works and catches real malformed XML
- No inline require() remains in the affected validation path
- The fix is compatible with the project’s ESM runtime
- Regression coverage is added so this cannot silently return
