#368 - Resolve all catalog TODO stubs in deterministic scaffold
What & Why
The deterministic scaffold's selectSystemActivity function has 6 explicit TODO stubs that emit LogMessage placeholders instead of mapping to real catalog activities. When a process references Google Calendar, Google Contacts, Gmail, GenAI, Document Understanding, or Integration Service, the scaffold produces useless "TODO: Configure…" log messages instead of actual activity XAML. This forces the AI enricher to improvise with generic HttpClient calls, manual JSON parsing, and extra variables — amplifying downstream errors.

Additionally, several enterprise systems (ServiceNow, Salesforce, Workday, Jira, SharePoint, Dynamics 365) fall back to generic HttpClient stubs with placeholder URLs. Where official UiPath activity packages exist for these systems (e.g., UiPath.Salesforce.Activities, UiPath.ServiceNow.Activities), the scaffold should map to real catalog activities instead of raw HTTP calls.

This task systematically resolves every TODO stub and upgrades enterprise system mappings to use real catalog activities where UiPath packages exist.
