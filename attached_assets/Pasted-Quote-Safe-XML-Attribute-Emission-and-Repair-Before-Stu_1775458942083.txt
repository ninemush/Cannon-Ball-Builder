Quote-Safe XML Attribute Emission and Repair-Before-Stub

What & Why
Repeated forensic evidence across recent bundles shows that many business workflows are being destroyed not because the business intent is absent, but because emitted XAML contains mechanically repairable XML well-formedness defects, especially raw quote characters inside expression-bearing attributes such as LogMessage.Message. The current pipeline often responds by stubbing or blocking the workflow instead of repairing the attribute value and continuing.

This task makes quote-safe emission a first-class requirement and changes the post-emission handling policy from “stub on repairable malformed-quote cases” to “repair deterministically before stubbing.”

Goals
1. Eliminate raw unescaped quote corruption in XML/XAML attribute values.
2. Repair mechanically fixable malformed-quote issues before any workflow is stubbed.
3. Preserve business workflows that are otherwise structurally sound.
4. Record clear before/after repair diagnostics for trust and traceability.

Done looks like
- No emitted XAML attribute contains raw quote characters that make the XML not well-formed.
- Expression-bearing attributes such as LogMessage.Message are safely serialized with proper XML escaping.
- If malformed-quote issues are detected post-emission and can be repaired deterministically, they are repaired in-place before any stub/block decision.
- Workflows are stubbed only when quote repair cannot restore a well-formed and semantically valid attribute.
- A structured diagnostics artifact records all quote repairs and all quote-repair failures.

Out of scope
- Broad mail-family/source-of-truth redesign
- Catalog/filter alignment
- General parity/freeze changes
- Deep business-logic invention
- Reworking the entire decomposition strategy

Tasks

1. Harden quote-safe attribute serialization
Audit all expression-bearing attribute emission paths, especially:
- LogMessage.Message
- Assign.Value when emitted as attribute
- other literal/vb_expression attribute serializers
Ensure raw quote characters inside serialized VB expressions are XML-escaped correctly.

2. Add pre-stub malformed-quote repair
Before any workflow is replaced with a stub due to malformed-quote or XML-wellformedness findings caused by quote corruption:
- attempt deterministic in-place quote repair
- re-run well-formedness check
- continue if repaired successfully
Only stub if repair fails or produces semantically unsafe output.

3. Restrict repair scope to safe deterministic cases
Repair only when the failure is clearly due to XML-escaping/quote corruption and the intended attribute value can be preserved exactly.
Do not guess business logic.

4. Emit diagnostics
Add:
quoteRepairDiagnostics
with per repair attempt:
- file
- attribute path
- original value
- repaired value
- repairApplied: true/false
- repairReason
- repairFailedReason
- packageFatal: true/false

Summary counts:
- totalMalformedQuoteFindings
- totalQuoteRepairsApplied
- totalQuoteRepairsFailed
- totalWorkflowsSavedFromStub
- totalWorkflowsStillStubbedAfterRepairAttempt

5. Integrate with package completeness / final validation
If quote repair succeeds, the workflow should continue through the normal pipeline without being stubbed for that defect class.
If quote repair fails, surface a clear structured defect instead of a vague stub outcome.

6. Regression tests
Cover:
- LogMessage.Message with embedded quotes is emitted as well-formed XML
- deterministic post-emission repair rescues a workflow that would previously be stubbed
- workflows are not stubbed for repairable quote-only defects
- unrecoverable malformed attributes still fail honestly
- repair diagnostics capture before/after values
