  Replace with context-aware handling:

  // replace with safe VB expression
  content = content.replace(
    /\[(?:[^\]]*?)(?:TODO_\w*|PLACEHOLDER_\w*)(?:[^\]]*?)\]/g,
    '[Nothing]'
  )
  
  // Outside expression context 
  // (plain attribute values)

The first replacement handles tokens 
inside [...] VB expression brackets — 
replaces the entire expression with 
[Nothing] which is valid VB for null.

The second replacement handles any 
remaining tokens outside brackets — 
replaces with empty string (not "").

This ensures no token ever produces 
malformed XML regardless of context.

FIX 3 — g flag bug at line 1293-1295

The fix-invalid-value regex uses the 
g flag for both .test() and .replace(). 
Calling .test() on a g-flagged regex 
advances lastIndex, causing the 
subsequent .replace() to potentially 
miss the first match.

Fix: Remove the .test() check entirely 
and just run .replace() unconditionally. 
If there are no matches, .replace() 
returns the original string unchanged 
with no performance cost worth worrying 
about. The .test() guard is unnecessary 
and actively harmful with the g flag.
