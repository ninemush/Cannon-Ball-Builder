How to simulate and test the connector changes (no code changes needed):

You already have a standalone comparison renderer script at scripts/compare-renderer.ts that uses your actual birthday automation process map data (25 nodes, 28 edges). Here's what to do:

Step 1 — Run the existing comparison script to see the current baseline:

npx tsx scripts/compare-renderer.ts
This outputs files to scripts/renderer-comparison/:

current.png — how the server-side renderer draws the map today
improved.png — a slightly improved version (larger nodes/fonts but same edge logic)
Step 2 — Have an agent create a new test script (e.g. scripts/test-orthogonal-connectors.ts) that:

Copies the existing compare-renderer.ts as a starting point — it already has all the infrastructure (dagre layout, SVG generation, sharp PNG export, the test node/edge data)

Modifies the edge rendering function to replace all bezier/curved paths with orthogonal step paths:

Every edge should use only horizontal and vertical line segments with rounded corners (quadratic bezier elbows at turns, radius ~12px)
From a bottom handle: go straight down, then elbow horizontally to align with target, then straight down to target
From a side handle: go straight horizontally, then elbow vertically to align with target, then straight to target
Increases decision branch displacement in the dagre config:

Set nodesep to at least 160 (currently 60/120) so Yes/No branch nodes have enough horizontal clearance for clean elbow connectors
Yes branches displaced left, No branches displaced right (the dagre rankdir: 'TB' handles this naturally with enough spacing)
Generates three output images for comparison:

baseline.png — current rendering (as-is, unchanged)
orthogonal-25-nodes.png — the same 25-node birthday process with orthogonal connectors
orthogonal-100-nodes.png — a synthetic 100-node map (duplicate the birthday flow 4x with unique IDs, connected end-to-end) to see how scaling looks
Outputs dimensions and file sizes so you can evaluate the 100-node impact on PDD images

Step 3 — Review the output PNGs in scripts/renderer-comparison/ to evaluate:

Are the connectors clean right-angled elbows?
Do decision branches have enough displacement?
Is the 100-node version still readable at 580px width (PDD scale)?
Are arrowheads visible at connector endpoints?
Key constraint for the test agent: Only create/modify files inside the scripts/ folder. Do not touch any application code in client/, server/, or shared/.
