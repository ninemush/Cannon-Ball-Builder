Post-Classifier Archive Freeze and Late-Stage Mutation Elimination

What & Why
The latest build no longer fails because of silent divergence in workflow classification logic alone. It now fails because archive bytes still change after the authoritative pre-archive classifier has already classified the archive-bound XAML.

The current failure proves there is at least one mutation path still running after classification:
- archive hash mismatches after pre-archive classification
- status changes after pre-archive classification
- parity checks now correctly catch this and stop the build

This task introduces a true freeze point for archive-bound XAML and workflow status. After that freeze point, no stage may mutate workflow bytes, inject stubs, inject placeholders, rewrite expressions, alter workflow status, or rewrite DHG-facing workflow content. Any such mutation must either move earlier in the pipeline or fail package mode loudly.

Goals
1. Eliminate all post-classifier mutations to archive-bound XAML and workflow status.
2. Introduce one explicit archive freeze point after canonicalization and authoritative classification.
3. Detect and fail any downstream mutation after that freeze point.
4. Preserve parity enforcement and make late mutations fully traceable.

Done looks like
- A single explicit archive freeze point exists after final canonicalization/mutation and authoritative workflow classification.
- After freeze, archive-bound XAML bytes are immutable.
- After freeze, workflow status classification is immutable unless the pipeline intentionally restarts classification from the new bytes.
- No stage after freeze may:
  - rewrite XAML
  - inject stubs
  - inject placeholders
  - alter workflow status
  - rewrite DHG-facing workflow content
  - rewrite deferredWrites/archive buffers
- A post-classifier mutation trace artifact is emitted showing whether any stage attempted to mutate frozen content.
- If any downstream stage mutates frozen archive-bound content or workflow status, package mode fails immediately with a clear stage-attributed error.
- Package-DHG parity and post-archive classifier verification remain enabled.

Out of scope
- Broad catalog/filter alignment
- New validator concepts
- Business-logic repair
- Reworking decomposition strategy
- Changing the high-level SDD/PDD generation flow

Tasks

1. Identify all post-classifier mutation paths
Trace every stage after authoritative workflow classification that can currently mutate:
- archive-bound XAML entries
- deferredWrites
- archive buffers
- workflow status
- DHG workflow input content
- package assembler workflow maps
Document each mutation point and whether it should move earlier or be forbidden.

2. Introduce an explicit archive freeze point
Create one explicit freeze step after:
- final canonicalization
- authoritative workflow classification
- final archive-bound XAML set preparation

At this point, persist:
- frozen workflow bytes
- frozen content hashes
- frozen workflow statuses
- frozen normalized filenames

3. Enforce immutability after freeze
After freeze, any stage that attempts to mutate frozen workflows or workflow statuses must:
- fail immediately in package mode
- emit a structured mutation violation
- include stage name, file, pre-hash, post-hash, mutation type, and reason

No silent mutation is allowed.

4. Move allowed mutations earlier
If a downstream stage still performs any of the following, move it to before freeze:
- stub injection
- placeholder injection
- expression repair
- quote repair
- final XAML normalization
- workflow status mutation
- DHG status inference mutation

If it cannot move earlier safely, package mode must fail instead of mutating after freeze.

5. Add post-classifier mutation trace artifact
Emit:
`postClassifierMutationTrace`
with per attempted mutation:
- stage
- file
- preHash
- postHash
- changedBytes
- changedStatus
- mutationReason
- allowed: true/false
- enforcementAction

Also emit a compact summary:
- totalAttemptedMutations
- totalBlockedMutations
- totalAllowedPreFreezeMutations
- totalPostFreezeViolations

6. Align DHG and archive to frozen truth
Ensure DHG generation, parity checks, and final reporting consume only the frozen workflow bytes and frozen workflow statuses.
No downstream component may re-read an earlier in-memory representation or recompute status from stale pre-freeze content.

7. Add regression tests
Cover:
- mutation attempt after freeze on workflow bytes triggers fatal failure
- mutation attempt after freeze on workflow status triggers fatal failure
- DHG generation from frozen content passes parity
- parity failure when a stale pre-freeze representation is intentionally used
- stub injection after freeze is blocked
- placeholder injection after freeze is blocked
- deferredWrites or archive buffer mutation after freeze is blocked
