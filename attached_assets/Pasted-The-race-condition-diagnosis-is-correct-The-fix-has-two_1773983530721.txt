The race condition diagnosis is correct. 
The fix has two parts:

FIX 1 — Remove the isStreaming guard 
from generateUiPath for the 
triggerUiPathGen path

The guard `if (isGenerating Doc || 
isStreaming) { return }` is correct 
for user-initiated clicks (prevent 
double-trigger) but wrong for the 
programmatic trigger path — when 
the server sends triggerUiPathGen, 
the chat stream is deliberately still 
in the process of closing.

Add a parameter to generateUiPath:

  generateUiPath(
    force: boolean = false
  )

When force is true, skip the 
isStreaming guard entirely. Only 
check isGeneratingDoc (prevent 
genuine double-generation).

Call it with force=true from the 
triggerUiPathGen handler:
  generateUiPath(true)

Call it with force=false (default) 
from button clicks and auto-triggers.

FIX 2 — Remove the setTimeout

The setTimeout is a symptom of the 
race condition, not a solution to it. 
Replace it with a direct call:

In handleSSEEvent, when 
triggerUiPathGen arrives:
  generateUiPathRef.current?.(true)

Call immediately, no setTimeout. 
The force=true parameter bypasses 
the stale isStreaming check so the 
timing doesn't matter.

ALSO — ensure generateUiPathRef 
always points to the latest version 
of generateUiPath. If generateUiPath 
is a useCallback with dependencies, 
the ref update must happen in a 
useEffect that runs when the callback 
changes:

  useEffect(() => {
    generateUiPathRef.current = 
      generateUiPath
  }, [generateUiPath])

This ensures the ref is never stale 
regardless of when triggerUiPathGen 
arrives.

COMPLETION CRITERIA

1. Trigger "regenerate uipath package" 
   via chat — confirm POST 
   /api/ideas/:id/generate-uipath 
   appears in server logs within 
   1 second of the chat SSE completing

2. Generation runs to completion 
   (success or failure on XML errors 
   is acceptable — what matters is 
   the endpoint is called)

3. No setTimeout in the trigger path

4. isStreaming guard is bypassed 
   for force=true calls only

5. All existing tests pass
