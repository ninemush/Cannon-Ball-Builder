## 2a. Agent Artifacts — Import and Configure

Agent artifacts have been generated and are included in this package.
They are NOT yet live in UiPath Agent Builder. Follow these steps in order.

### What's in the package

| File | What it is |
|------|------------|
| prompts/system_prompt.txt | Generated system prompt — review [REVIEW:] tags before use |
| prompts/user_prompt_template.txt | Parameterized prompt template for runtime inputs |
| tools/tool_definitions.json | Tool definitions — verify schemas match your system APIs |
| knowledge/kb_placeholder.md | List of documents to upload and index in your knowledge base |
| agents/[AgentName]_config.json | Full agent config ready to import into Agent Builder |

### Import steps

1. Open UiPath Agent Builder in your tenant
2. Create a new agent named [AgentName]
3. Import agents/[AgentName]_config.json as the agent configuration
4. Upload and index the documents listed in knowledge/kb_placeholder.md
5. Authorize any OAuth connections flagged with # AUTHORIZE: in the config

### Configure checklist

| # | File | Action |
|---|------|--------|
| 1 | system_prompt.txt | Validate all [REVIEW:] sections against your real process |
| 2 | [AgentName]_config.json | Complete all AUTHORIZE flags with real connection credentials |
| 3 | [AgentName]_config.json | Tune all TUNE flags (temperature, iterations) against test runs |
| 4 | tool_definitions.json | Verify tool schemas match actual API responses from your systems |
| 5 | knowledge/kb_placeholder.md | Upload all listed documents before first agent run |

### What works out of the box
- Agent structure, tool routing, and escalation logic
- System prompt (generated from your actual process narrative)
- RPA sequences and RPA-to-Agent handoff points

### What requires human action before the agent will run
- OAuth connection authorization (cannot be done programmatically)
- Knowledge base document upload and indexing
- Production prompt validation against real data

What to leave completely unchanged

Deployment pipeline and all API publishing logic
RPA workflow generation
Deployment Report structure, versioning, counts, and all RPA artifact statuses
Config.xlsx generation
Tier 1 / Tier 2 / Tier 3 structure and readiness score
The existing download button and zip behavior (just add the new files to the zip)


The only visible changes to the user are: amber "In Package" badges on agent artifacts in the Deployment Report, the updated bottom message, and the richer package contents when they download.
