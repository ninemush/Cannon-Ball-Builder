PROCESS DESIGN DOCUMENT (PDD)
After the As-Is map is approved, the LLM should automatically generate a PDD in the chat. 

Trigger this by sending the following to Claude after approval, along with the full conversation history and a JSON snapshot of all current process map nodes:

"The SME has approved the As-Is process map. Now generate a Process Design Document. The PDD must include: 1) Executive Summary, 2) Process Scope, 3) As-Is Process Description (narrative form, referencing the steps in the map), 4) Pain Points and Inefficiencies, 5) Automation Opportunity Assessment, 6) Assumptions and Exceptions, 7) Data and System Requirements. Write this as a professional document, not a bullet list. Be specific and use the details from our conversation."

RENDERING THE PDD IN CHAT
Render the PDD response in the chat with:
- A distinct card style — slightly elevated surface (#2a2a2a), left border in teal, a document icon header
- Sections are collapsible
- An "Approve PDD" button at the bottom of the card
- The SME can continue chatting to request changes. If they say something like "update section 3" or "add an exception for X", send the updated request to Claude and re-render the PDD card with the revision (keep the old version in history, show the new one as current)
- Approval flow same as As-Is map: confirmation modal → write to approvals table → chat acknowledgment → stage advances

Parse the JSON response and:
