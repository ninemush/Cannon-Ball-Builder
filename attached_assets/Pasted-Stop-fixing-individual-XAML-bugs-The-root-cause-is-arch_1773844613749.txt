Implement hierarchical generation in three passes:

PASS 1 — Tree-structured decomposition

Change the SDD decomposition output from a flat step 
list to a typed workflow tree. Each node must explicitly 
declare its parent-child relationships.

Define this TypeScript structure:

type WorkflowNode =
  | ActivityNode      // leaf — a single activity
  | TryCatchNode      // structural — has try/catch children
  | IfNode            // structural — has then/else children
  | SequenceNode      // structural — has ordered children
  | RetryScopeNode    // structural — has body children

type ActivityNode = {
  kind: 'activity'
  template: string        // exact template name from catalog
  displayName: string
  properties: Record<string, string>  // placeholder values
  errorHandling: boolean  // wrap in standard TryCatch
}

type TryCatchNode = {
  kind: 'trycatch'
  displayName: string
  errorNote: string
  tryChildren: WorkflowNode[]
  // catch block is always the standard error handler template
  // no custom catch children needed
}

type IfNode = {
  kind: 'if'
  displayName: string
  condition: string
  thenChildren: WorkflowNode[]
  elseChildren: WorkflowNode[]
}

type SequenceNode = {
  kind: 'sequence'
  displayName: string
  children: WorkflowNode[]
}

type RetryScopeNode = {
  kind: 'retryScope'
  displayName: string
  numberOfRetries: number
  retryInterval: string   // HH:MM:SS format
  bodyChildren: WorkflowNode[]
}

type WorkflowSpec = {
  name: string
  variables: VariableDeclaration[]
  root: SequenceNode
}

type VariableDeclaration = {
  name: string
  type: string   // x:String, x:Boolean, x:Int32 etc
  default?: string
}

The decomposition LLM call produces a JSON object 
matching WorkflowSpec. Not prose. Not step lists. 
A typed tree.

The decomposition prompt must instruct the LLM:
"Output a JSON object matching the WorkflowSpec type. 
Every activity that can fail must be a child of a 
TryCatchNode, not a sibling. Never place activities 
as direct children of the root Sequence unless they 
genuinely have no error handling. The catch block is 
always the standard error handler — do not add custom 
catch content."

PASS 2 — Leaf node template resolution (no LLM)

For each ActivityNode in the tree, call 
xamlTemplateBuilder.resolveActivity(node) which:
1. Looks up the template by node.template name
2. Fills in each {{placeholder}} from node.properties
3. If node.errorHandling is true, wraps the activity 
   in the standard TryCatch structural template
4. Returns the complete XML string for that leaf

This is deterministic code. No LLM involvement.

For structural nodes (TryCatchNode, IfNode, etc), 
the templates are fixed — they just need their 
child XML inserted into the correct slots.

PASS 3 — Bottom-up tree assembly (no LLM)

Walk the WorkflowSpec tree bottom-up:

function assembleNode(node: WorkflowNode): string {
  if (node.kind === 'activity') {
    return xamlTemplateBuilder.resolveActivity(node)
  }
  if (node.kind === 'trycatch') {
    const tryXml = node.tryChildren
      .map(assembleNode).join('\n')
    return standardTryCatchTemplate
      .replace('{{tryActivities}}', tryXml)
      .replace('{{displayName}}', node.displayName)
      .replace('{{errorHandlingNote}}', node.errorNote)
      // ... fill other placeholders
  }
  if (node.kind === 'if') {
    const thenXml = node.thenChildren
      .map(assembleNode).join('\n')
    const elseXml = node.elseChildren
      .map(assembleNode).join('\n')
    return ifTemplate
      .replace('{{thenActivities}}', thenXml)
      .replace('{{elseActivities}}', elseXml)
      .replace('{{condition}}', node.condition)
      // ... fill other placeholders
  }
  if (node.kind === 'sequence') {
    const childXml = node.children
      .map(assembleNode).join('\n')
    return sequenceTemplate
      .replace('{{activities}}', childXml)
      // ... fill other placeholders
  }
  if (node.kind === 'retryScope') {
    const bodyXml = node.bodyChildren
      .map(assembleNode).join('\n')
    return retryScopeTemplate
      .replace('{{retryBodyActivities}}', bodyXml)
      // ... fill other placeholders
  }
}

The variable declarations are built programmatically 
from the WorkflowSpec.variables array — not generated 
by the LLM. Every variable in the array becomes a 
Variable element in Sequence.Variables. The LLM 
populates the variables array in the decomposition 
output; the system writes the XML.

Wrap the root sequence with the fixed workflow header 
to produce the complete XAML file.
